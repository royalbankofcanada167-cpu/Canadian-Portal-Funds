📐 Complete Prisma Schema

The following is the complete, production-ready Prisma schema configuration:

// Prisma Schema File: schema.prisma
// Generated for Tshore Bank Database

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ============================================
// USER MODEL
// ============================================
// Represents system users and account holders
model User {
  id         Int        @id @default(autoincrement())
  email      String     @unique
  password   String
  createdAt  DateTime   @default(now())

  // Relations
  wallets    Wallet[]
  transfers  Transfer[] @relation("senderTransfers")
  received   Transfer[] @relation("receiverTransfers")
}

// ============================================
// WALLET MODEL
// ============================================
// Represents user financial wallets with balance tracking
model Wallet {
  id         Int       @id @default(autoincrement())
  userId     Int
  balance    Decimal   @default(0)
  currency   String    @default("CAD")
  createdAt  DateTime  @default(now())

  // Relations
  user       User      @relation(fields: [userId], references: [id])

  @@index([userId])
}

// ============================================
// TRANSFER MODEL
// ============================================
// Represents financial transfers between users
model Transfer {
  id          Int       @id @default(autoincrement())
  amount      Decimal
  senderId    Int
  receiverId  Int
  createdAt   DateTime  @default(now())

  // Relations
  sender      User      @relation(
                "senderTransfers",
                fields: [senderId],
                references: [id]
              )

  receiver    User      @relation(
                "receiverTransfers",
                fields: [receiverId],
                references: [id]
              )

  @@index([senderId])
  @@index([receiverId])
}
📌 Note: This schema includes comprehensive indexing on foreign key columns for optimal database performance. All timestamps use UTC timezone. Decimal type is used for currency values to prevent floating-point precision issues.

📊 Data Models

👤 User Model

Represents a registered user in the Tshore Bank system. Each user can own multiple wallets and initiate or receive transfers. User authentication is handled through email and encrypted password storage.

id Int
Primary key, auto-incremented integer identifier
email String
Unique email address for user authentication and identification
password String
Hashed password for secure authentication (must be encrypted before storage)
createdAt DateTime
Account creation timestamp, automatically set to current time

🔗 Relationships

wallets: One-to-many relationship with Wallet model. One user can have multiple wallets.
transfers (senderTransfers): One-to-many relationship representing transfers sent by the user.
received (receiverTransfers): One-to-many relationship representing transfers received by the user.

💰 Wallet Model

Represents a financial wallet belonging to a user. Users can maintain multiple wallets for different currencies. Each wallet tracks a balance and currency type with creation timestamp.

id Int
Primary key, auto-incremented integer identifier
userId Int
Foreign key referencing the User model owner of this wallet
balance Decimal
Current wallet balance (defaults to 0, stored as high-precision decimal)
currency String
Currency code in ISO 4217 format (defaults to "CAD" - Canadian Dollar)
createdAt DateTime
Wallet creation timestamp, automatically set to current time

🔗 Relationships

user: Many-to-one relationship with User model. Each wallet belongs to exactly one user.

📇 Indexes

userId
Index on userId for efficient queries of wallets by user

💸 Transfer Model

Represents a monetary transfer between two users. Each transfer is immutable once created and includes timestamp tracking for audit purposes. Supports peer-to-peer transactions with sender and receiver verification.

id Int
Primary key, auto-incremented integer identifier
amount Decimal
Transfer amount in decimal format (high-precision for currency accuracy)
senderId Int
Foreign key referencing the User model (sender of the transfer)
receiverId Int
Foreign key referencing the User model (receiver of the transfer)
createdAt DateTime
Transfer execution timestamp, automatically set to current time

🔗 Relationships

sender: Many-to-one relationship with User model via senderTransfers alias.
receiver: Many-to-one relationship with User model via receiverTransfers alias.

📇 Indexes

senderId
Index on senderId for efficient queries of outgoing transfers
receiverId
Index on receiverId for efficient queries of incoming transfers

✨ Key Features & Best Practices

🔍 Data Integrity

Unique Constraints: Email addresses are unique to prevent duplicate user accounts
Foreign Key Relationships: All references maintain referential integrity
Decimal Precision: Currency values use Decimal type for exact financial calculations
Timestamps: All records include creation timestamps for audit trails

⚡ Performance Optimization

Strategic Indexing: Indexes on foreign key columns (userId, senderId, receiverId) improve query performance
Query Optimization: Use Prisma's relation loading to avoid N+1 queries
Connection Pooling: Configure PostgreSQL connection pooling for high-throughput scenarios

🔐 Security Considerations

Password Hashing: Always hash passwords using bcrypt or similar before storing
Input Validation: Validate all user inputs on the application layer
Transaction Verification: Implement validation to prevent invalid transfers
Access Control: Implement role-based access control at the application level

🚀 Setup Instructions

1️⃣ Installation

npm install @prisma/client
npm install -D prisma

2️⃣ Environment Configuration

Create a .env file in your project root:

DATABASE_URL="postgresql://user:password@localhost:5432/tshore_bank"

3️⃣ Database Setup

# Create migration
npx prisma migrate dev --name init

# Apply migrations
npx prisma migrate deploy

# Generate Prisma Client
npx prisma generate

4️⃣ Verify Installation

# Open Prisma Studio to inspect database
npx prisma studio

💡 Usage Examples

Create User

const user = await prisma.user.create({
  data: {
    email: "john@example.com",
    password: hashedPassword,
  },
});

Create Wallet

const wallet = await prisma.wallet.create({
  data: {
    userId: user.id,
    balance: 1000.00,
    currency: "CAD",
  },
});

Execute Transfer

const transfer = await prisma.transfer.create({
  data: {
    amount: 250.50,
    senderId: user1.id,
    receiverId: user2.id,
  },
});

Fetch User with Relations

const userWithData = await prisma.user.findUnique({
  where: { id: userId },
  include: {
    wallets: true,
    transfers: true,
    received: true,
  },
});
